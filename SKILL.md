## 核心理念

### 1. 赔率优先原则

```
赔率（RR）永远放在第一位
最小RR = 2.0（输了亏1%，赢了至少赚2%）
胜率是第二优先级
```

### 2. 动态杠杆

| 市场环境 | 趋势强度 | 置信度 | 杠杆 |
|---------|---------|--------|------|
| 强趋势 | ADX≥70 | ≥0.8 | 3x |
| 中趋势 | ADX≥50 | ≥0.6 | 2x |
| 弱趋势 | ADX<50 | <0.6 | 1x |
| 不明确 | - | - | 禁止开仓 |

### 3. 科学仓位

```
基础仓位 = 账户2%
实际仓位 = 基础仓位 × 杠杆系数 × 置信度系数

杠杆系数：1x=1.0, 2x=1.5, 3x=1.8
置信度系数：0.5（低）~ 1.0（高）

最大单币仓位：15%
最大总仓位：40%
```

### 4. 多因子信号

| 因子 | 权重 | 数据来源 |
|------|------|---------|
| 价格动量 | 60% | RSI/ADX/MACD |
| 新闻情感 | 20% | 新闻API |
| 链上数据 | 10% | 交易所流量/大额转账 |
| 钱包分布 | 10% | 持币集中度 |

## 快速开始

### 1. 安装依赖

```bash
cd ~/Desktop/miracle-1.0.1
pip install -r requirements.txt
```

### 2. 配置API密钥

编辑 `miracle_config.json`：
```json
{
  "api": {
    "okx": {
      "api_key": "your_okx_api_key",
      "secret_key": "your_okx_secret",
      "passphrase": "your_passphrase"
    },
    "binance": {
      "api_key": "your_binance_api_key",
      "secret_key": "your_binance_secret"
    }
  }
}
```

### 3. 测试运行

```bash
# 单次扫描
python agents/agent_coordinator.py --symbol BTC

# 扫描所有币种
python agents/agent_coordinator.py

# 查看状态
python agents/agent_coordinator.py --status

# 持续运行（每30分钟）
python agents/agent_coordinator.py --daemon
```

## 交易规则

### 入场条件

| 方向 | 条件 |
|------|------|
| **做多** | RSI<45 + ADX>30 + 新闻偏利好 + 链上净流入 |
| **做空** | RSI>65 + ADX>30 + 新闻偏利空 + 链上净流出 |
| **观望** | ADX<25 或 因子矛盾 |

### 出场规则

| 类型 | 规则 |
|------|------|
| **止损** | max(3xATR, 账户1%风险) |
| **止盈** | RR≥2.0（赔率保证） |
| **时间止损** | 持仓超过24小时强制平仓 |
| **保本** | 盈利达到3xATR时移动止损到保本 |

### 熔断规则

| 条件 | 操作 |
|------|------|
| 单日亏损>5% | 停止交易1天 |
| 连续亏损2笔 | 停止交易1天 |
| 最大回撤>20% | 停止交易，等待人工审查 |

### 高频限制

```
最小交易间隔：2小时
同一方向最多连续：3笔
日交易上限：5笔
```

## 监控指标

### 每日监控

| 指标 | 阈值 | 告警 |
|------|------|------|
| 日胜率 | <40% | ⚠️ 检查 |
| 日均RR | <1.5 | 🔴 告警 |
| 单日亏损 | >5% | 🔴 熔断 |
| 最大回撤 | >15% | 🔴 告警 |

### 因子有效性

| IC值 | 含义 | 操作 |
|------|------|------|
| IC<0.02 | 无效 | 权重降50% |
| IC 0.02-0.05 | 弱有效 | 降低仓位50% |
| IC 0.05-0.10 | 有效 | 正常 |
| IC>0.10 | 强有效 | 全额 |

## 学习机制

### 自学习规则

1. **因子权重调整**
   - 因子IC持续<0.02超过5天 → 权重降低50%
   - 因子IC>0.05持续5天 → 权重恢复

2. **模式淘汰**
   - 胜率<40% → 移出白名单
   - 平均RR<1.5 → 移出白名单

3. **模式新增**
   - 连续5笔盈利 + 胜率>50% → 加入白名单

### 报告周期

| 周期 | 内容 |
|------|------|
| 每日 | 交易统计、盈亏摘要 |
| 每周 | 因子表现、策略调整建议 |
| 每月 | 全面复盘、参数优化 |

## 与 Kronos 的区别

| 方面 | Kronos | Miracle 1.0.1 |
|------|--------|--------------|
| **频率** | 低频（每日1-2笔） | 高频（每日至少1笔） |
| **信号** | 纯价格因子 | 多因子（价格+新闻+链上+钱包） |
| **杠杆** | 固定1-2x | 动态1-3x |
| **学习** | 手动 | 自动 |
| **仓位** | 固定比例 | 科学计算（置信度加权） |
| **目标** | 稳健 | 赔率优先 |

## 注意事项

⚠️ **高风险**：高频+杠杆系统，请务必：
1. 先用模拟盘验证至少3个月
2. 小资金起步（<5%总资金）
3. 严格遵守熔断规则
4. 定期审查策略表现
